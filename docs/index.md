# 欢迎来到东华高中毕业生大学指北 🎓

## 📖 文档作用

这是一个由东华高中毕业生共同维护的**大学信息分享平台**。我们收集整理了各地知名高校的真实就读体验、专业介绍、校园生活、报考建议等内容，旨在为学弟学妹们的升学选择提供第一手参考资料。

通过这份指南，你可以收获到包括但不限于：

- **了解真实的大学生活** - 学长学姐的亲身经历和感受
- **获取专业信息** - 各专业的课程设置、学习难度、就业方向
- **掌握报考技巧** - 录取分数线、招生政策、志愿填报建议
- **规划未来发展** - 深造、就业、出国等多元路径参考

## ⚠️ 免责声明

1. **内容仅供参考**：本站所有内容均来自东华高中毕业生的个人经验分享，代表作者个人观点，不代表学校官方立场。
2. **信息时效性**：大学政策、专业设置、录取标准等会随时间变化，请以各高校官方网站的最新信息为准。
3. **个体差异性**：每个人的体验和感受因个人背景、兴趣、能力而异，请结合自身情况理性参考。
4. **非官方平台**：本站为学生自发维护的非盈利性质文档，与各高校无正式关联。
5. **作者水平有限**：本站出现的任何 bug 均来自作者的水平不足，请多多见谅。

## 👀 阅读者使用方法

这一部分为学弟学妹们提供使用本指北书的使用方法。

### 快速导航

通过顶部导航栏选择你感兴趣的地区，查看对应高校的信息，比如：

- **山东** - 山东大学
- **广东** - 中山大学
- **湖北** - 武汉大学
- **甘肃** - 兰州大学

### 搜索功能

点击页面右上角的 🔍 搜索图标，输入关键词快速查找内容，比如：

- 搜索学校名：如"北京大学"
- 搜索专业名：如"计算机"
- 搜索关键词：如"食堂"、"宿舍"

### 阅读建议

1. **对比多所学校** - 不要只看一所大学，建议横向对比
2. **关注多个维度** - 学术、生活、就业等多方面了解
3. **保持批判思维** - 理性看待内容，结合自身判断
4. **查看官方信息** - 重要决策前务必访问学校官网确认

## ✍️ 编辑者使用方法

我们欢迎所有东华高中的学长学姐参与内容贡献！

### 内容规范

- **真实性优先**：分享真实体验，避免夸大或误导
- **结构化组织**：使用标题、列表等让内容清晰易读
- **保护隐私**：不要透露个人敏感信息或他人隐私
- **言论合法**：保持客观理性，避免攻击性言论、辱骂性言论，避免涉及任何带有政治的内容
- **Markdown 格式**：使用标准 Markdown 语法编写

### 文档结构

以山东大学为例，任何有关学校的**客观情况**，请填入**学校概况**部分；如果想要补充新的内容，请在对应学校的文件夹下面创建新的 markdown 文档，使用英文或者中文拼音命名。

```
docs/
├── shandong/
│   └── sdu/
│       ├── sdu.md    # 学校概况
```

如果是其他与大学相通的内容，请放在**其他**部分，如：

```
docs/
├── others/
│   ├── a.md    # 高中与大学的区别
```

如果您发现您编写的内容与文档中已经存在的内容有重复之处，也可以上传，作者会使用文档的署名区分开来。

作者非常欢迎海外大学的学长学姐参与文档的编写！

### 贡献流程

#### 方式一：发送邮箱

对于完全不会 markdown 文档编写的学长学姐，推荐使用这种方式。

将你们想要写的内容发送到邮箱 University_Guide@163.com ，请在主题上写“东华高中毕业生大学指北投稿”，署名务必加上自己的学校以及您的年级。

#### 方式二：本地开发（推荐熟悉 Git 的同学）

```bash
# 1. Fork 项目到你的 GitHub 账号
# 访问：https://github.com/musemok/university-guide
# 点击右上角的 Fork 按钮

# 2. 克隆你 Fork 后的仓库到本地
git clone https://github.com/你的用户名/university-guide.git
cd university-guide

# 3. 创建虚拟环境并安装依赖
python -m venv venv
venv\Scripts\activate  # Windows
pip install -r requirements.txt

# 4. 本地预览
mkdocs serve
# 访问 http://127.0.0.1:8000 预览效果

# 5. 创建新分支进行编辑（推荐）
git checkout -b add-pku-life  # 使用描述性名称命名分支

# 6. 编辑内容
# 在 docs/ 目录下编辑或新增 Markdown 文件
# 例如：docs/beijing/pku/life.md

# 7. 更新导航配置（重要！）
# 编辑 mkdocs.yml，在 nav 部分添加新文档
# 例如：
#   - 北京:
#       - 北京大学:
#           - 学校概况: beijing/pku/overview.md
#           - 校园生活: beijing/pku/life.md  # 新增这一行

# 8. 刷新预览查看效果
# mkdocs serve 会自动重载，检查导航是否正确显示

# 9. 提交更改到你的分支
git add .
git commit -m "添加北京大学校园生活介绍"
git push origin add-pku-life  # 推送到你 Fork 的仓库

# 10. 提交 Pull Request
# 访问你的 GitHub 仓库页面：https://github.com/你的用户名/university-guide
# 会看到 "Compare & pull request" 按钮，点击它
# 填写 PR 标题和说明，描述你添加的内容
# 提交后等待维护者审核
```

### 审核流程

- 所有贡献内容需要通过 **Pull Request** 提交
- 维护者会审核内容的**真实性、规范性和准确性**
- 审核通过后才会合并到 `main` 分支并自动发布
- 通常会在 **1-3 个工作日**内完成审核反馈
